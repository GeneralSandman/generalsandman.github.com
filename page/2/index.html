Linux多线程编程（一）概念

发表于 2017-01-04 | 分类于 Linux多线程编程 |

一、线程与进程

1）区别

进程和线程的主要差别在于它们是不同的操作系统资源管理方式：

进程有独立的地址空间，一个进程崩溃后，在保护模式下不会对其它进程产生影响，而线程只是一个进程中的不同执行路径。线程有自己的堆栈和局部变量，但线程之间没有单独的地址空间，一个线程死掉就等于整个进程死掉，所以多进程的程序要比多线程的程序健壮，但在进程切换时，耗费资源较大，效率要差一些。但对于一些要求同时进行并且又要共享某些变量的并发操作，只能用线程，不能用进程。

2）优缺点

线程执行开销小，但不利于资源的管理和保护；而进程正相反。

二、如何选择线程和进程

###线程的优点
1.线程之间数据共享非常简单。相比之下，进程之间的数据共享需要更多的投入。(例如创建共享内存段或者使用pipe管道)
2.创建线程快于创建进程。线程之间的上下文切换消耗的时间比较短。

###进程的优点
1.多线程编程时，要确保调用线程安全的函数，或者以线程安全的方式调用函数，然而多进程不要关注这些(由操作系统来控制进程安全)。
2.某个线程的bug，肯能会影响其他线程，因为线程之间共享着相同的地址空间。相反，进程之间的数据隔离更加彻底。
3.每个线程都在争用宿主进程的有限地址空间。如果线程栈和线程特有数据消耗掉进程虚拟地址空间，后续线程将无法使用这些区域。
4.不利于资源的管理和保护。

###选择进程线程注意事项
1.除了数据，线程之间还会共享：文件描述符、信号处理、当前工作目录、用户ID和组ID等等。

Hexo更改主题

##第一步：选择主题
hexo含有两份配置文件(_config.yml)，一份位于根目录下——站点配置文件，一份位于主题目录下——主题配置文件。
hexo安装主题只需要把主题文件放置到根目录下的themes目录下，然后修改配置文件即可。具体安装步骤如下：

###1）下载next主题
通过git将next主题下载到themes文件next下

>git clone https://github.com/iissnan/hexo-theme-next shemes/next

###2）启动主题
下载完成后，找到站点配置文件，找到theme字段，改为theme： next
注意：冒号后有空格

###3）验证主题是否运行
运行

hexo s debug

访问

http://localhost:4000

如果4000端口被占用了，出现错误

FATAL Port 4000 has been used. Try other port instead.
hexo server -p 5000
更改端口测试

确保站点正确运行

##第三步：配置

###选择Scheme
next通过scheme选择主题中的主题。Mist是NexT的第一款主题。启动Mist需要将主题配置文件中的
-#scheme:Mist 前面的注释#去掉

###语言设置
编辑站点配置文件，将language设置成所需要的语言即可

语言 设定值
English language: en
简体中文 language: zh-Hans
正体中文 language: zh-hk

###菜单设置
配置位于主题配置文件menu

键值 设定值 comment
home home: / 主页
archives archives: /archives 归档页
categories categories: /categories 分类页（需手动创建）
tags tags: /tags 标签页（需手动创建）
about about: /about 关于页面 （需手动创建）
commonweal commonweal: /404.html 公益 404 （需手动创建

###侧边设置
默认情况下，侧栏仅在文章页面（拥有目录列表）时才显示。可以通过修改 主题配置文件 中的 sidebar 字段来控制侧栏的行为。

支持的选项有：

post - 默认行为，在文章页面（拥有目录列表）时显示
always - 在所有页面中都显示
hide - 在所有页面中都隐藏（可以手动展开）

###头像设置
编辑站点配置文件，新增字段avatar，值设置为链接地址。
头像的链接地址可以是：

地址 值
完整的互联网URL https://avatars1.githubusercontent.com/u/32269?v=3&s=460
站点内的地址 /uploads/avatar.jpg - 需要将你的头像图片放置在 站点的 source/uploads/（可能需要新建uploads目录）

###作者名称
编辑 站点配置文件，设置 author 为你的昵称。

###站点描述设置
编辑 站点配置文件，设置 description 字段为你的站点描述。站点描述可以是你喜欢的一句签名:)
